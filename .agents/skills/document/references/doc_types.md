# Doc Types

Use these as default structures when the user does not already have a template.

## Decision Doc

Recommended sections:

1. Summary
2. Context
3. Problem
4. Decision
5. Options considered
6. Tradeoffs
7. Rollout or follow-up

## Technical Spec

Recommended sections:

1. Summary
2. Goals
3. Non-goals
4. Current state
5. Proposed design
6. Edge cases
7. Risks
8. Testing and rollout

## Proposal

Recommended sections:

1. Summary
2. Problem or opportunity
3. Proposal
4. Expected impact
5. Cost and constraints
6. Risks
7. Next steps

## RFC

Recommended sections:

1. Summary
2. Motivation
3. Detailed design
4. Alternatives
5. Migration or rollout
6. Open questions

## Runbook

Recommended sections:

1. Purpose
2. Preconditions
3. Step-by-step procedure
4. Verification
5. Failure modes
6. Escalation

## PRD

Recommended sections:

1. Summary
2. Problem
3. Users
4. Goals
5. Requirements
6. Non-goals
7. Metrics
8. Risks and dependencies
