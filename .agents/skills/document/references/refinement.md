# Refinement

Use this loop for each section:

1. Ask what the section must help the reader do or understand.
2. Generate 5-10 candidate points for the section.
3. Ask the user what to keep, cut, or merge.
4. Draft the section.
5. Remove repetition, hedging, and generic filler.

Signals that a section is still weak:

- it references background that is not explained elsewhere
- it names a decision without the reason behind it
- it mixes summary and details in the same paragraph
- it repeats points already made in another section
- it assumes the audience already agrees with unstated premises

When a section is stable, ask one final question:

- What can be deleted here without harming the reader?
